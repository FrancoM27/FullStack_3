DOCUMENTACION TECNICA - GAMEBAKES
==================================

RESUMEN EJECUTIVO
-----------------

Gamebakes es una plataforma de comercio electronico de productos horneados implementada con arquitectura de microservicios. La aplicacion permite a los usuarios registrarse, navegar productos, realizar compras, dejar reseñas y gestionar sus perfiles, mientras que los vendedores pueden gestionar su inventario y procesar pedidos.

CARACTERISTICAS PRINCIPALES:
- Gestion de Usuarios: Registro, login, recuperacion de contrasena y perfiles
- Catalogo de Productos: Visualizacion, filtrado y busqueda de productos
- Sistema de Pedidos: Carrito de compras, seguimiento de estado de pedidos
- Sistema de Reseñas: Calificacion y comentarios sobre productos
- Pagos: Integracion con Mercado Pago
- Roles: CLIENTE y VENDEDOR con permisos diferenciados

ARQUITECTURA:
- Frontend: React 19 + Vite
- Backend: 7 microservicios Spring Boot + API Gateway
- Infraestructura: Docker Compose
- Base de Datos: MySQL 8.0
- Mensajeria: Apache Kafka
- Autenticacion: JWT tokens

- /api/usuarios/** -> servicio-usuarios:8080

- /api/productos/** -> servicio-productos:8085
- /api/pedidos/** -> servicio-pedidos:8082
- /api/resenas/** -> servicio-resenas:8081

FLUJO DE DATOS COMPLETO
------------------------

1. FLUJO DE REGISTRO Y LOGIN

Usuario -> Frontend (Login.jsx)
         POST /api/usuarios/login
         {email, password}
         API Gateway (AuthenticationFilter)
         (sin token, ruta publica)
         Servicio Usuarios
         Validar credenciales
         Generar JWT token
         Frontend (guarda token en sessionStorage)
         Usuario autenticado


2. FLUJO DE COMPRA DE PRODUCTO

Usuario -> Frontend (CatalogoProductos.jsx)
         Selecciona producto
         Agrega al carrito
         Frontend (Carrito.jsx)
         POST /api/pagos/crear-preferencia
         Headers: Authorization: Bearer <token>
         API Gateway (AuthenticationFilter)
         Valida JWT
         Agrega X-User-Id, X-User-Role
         Servicio Pagos
         Crea preferencia Mercado Pago
         Retorna URL de pago
         Frontend -> Redireccion a Mercado Pago
         Usuario completa pago
         Mercado Pago -> Webhook /api/pagos/webhook
         Servicio Pagos
         Procesa pago exitoso
         Reduce stock (Feign -> Servicio Productos)
         Publica evento Kafka (pagos-exitosos)
         Kafka -> Servicio Pedidos (PedidoListener)
         Crea pedido
         Notifica al usuario


3. FLUJO DE RESENA

Usuario -> Frontend (ResenasProducto.jsx)
         POST /api/resenas
         Headers: Authorization: Bearer <token>
         {productoId, comentario, estrellas}
         API Gateway (AuthenticationFilter)
         Valida JWT
         Agrega X-User-Id, X-User-Role
         Servicio Reseñas
         Valida compra (Feign -> Servicio Pedidos)
         Valida entregado (Feign -> Servicio Pedidos)
         Si valido: guarda reseña
         Publica evento Kafka
         Frontend -> Actualiza lista de reseñas


4. FLUJO DE GESTION DE PRODUCTOS (VENDEDOR)

Vendedor -> Frontend (GestionProductos.jsx)
          POST /api/productos
          Headers: Authorization: Bearer <token>
          {nombre, descripcion, precio, categoria}
          API Gateway (AuthenticationFilter)
          Valida JWT
          Verifica rol = VENDEDOR
          Agrega X-User-Id, X-User-Role
          Servicio Productos
          Crea producto
          Asocia al vendedor (X-User-Id)
          Frontend -> Actualiza catalogo

CONCLUSION
----------

Gamebakes implementa una arquitectura de microservicios moderna y escalable con separacion clara de responsabilidades. El uso de API Gateway centraliza la autenticacion y enrutamiento, mientras que Kafka permite comunicacion asincronica entre servicios. El frontend React proporciona una experiencia de usuario fluida con gestion de estado local y autenticacion basada en JWT.

La arquitectura esta preparada para escalar horizontalmente, con cada microservicio capaz de operar independientemente. La implementacion actual cubre los casos de negocio principales y proporciona una base solida para futuras mejoras.
